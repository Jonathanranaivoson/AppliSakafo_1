PROJET APPLICATION WEB : APPLICATION DE GESTION DES MENUS JOURNALIERE DANS UN FOYER DOMESTIQUES
Format : Java/Jsp
Base de donnees : PostgreSQL

------------------------------
1. INTITULEE ET SUJET:
* Plan menu = ensemble d'un ou plusieur menus

 a) Application permettant de planifier et gerer les menus alimentaires journalier dans un foyer domestiques
     |-> Plannifier, Gerer et prevoir les courses a faire si dans le BESOINS
 b) Regle en generale:
     * Petit-dejeuner: 
          -> repartit comme suit: _ groupe 1 (pour les riz + met)
                                  _ groupe 2 (extra ou les noms riz(pain, Cereales, ...))
                                  _ Boisson
          -> Petit-dejeuner = menu(issu groupe 1 ou 2) + Boisson
     * Dejeuner:
          -> repartit comme suit: _ 4 groupe de 6 menus chacune (24 menu en tout) sauf dimanche
     * Diner:
          -> repartit comme suit: _ 4 groupe de 6 menus chacune (24 menu en tout) sauf dimanche
          -> le diner contient un autre groupe nommee extra contenant d'autres menus interchangeable avec les menus du diner
 c) L'Application se voit comme suit:
      * Chaque jour = 1 plan menu -> Un Petit dejeuner + Un Dejeuner + Un Diner
      * Chaque semaine = 6 menus differente (dimanche non inclus)
      * Chaque menu peut etre interchangee avec d'autre menu
      * Vu la regle des 4 groupes de menu hebdomadaire(contenant 6 menu journaliere (dejeuner et diner)) , les menus peut revenir apres fin de cycle:
          -> ex: semaine 1 = groupe 1
                 semaine 2 = groupe 2
                 ... le cycle revient lorsque semaine 4 = groupe 4 est terminee
      * Le Petit-dejeuner etant non divisee semblable a celui du Dejeuner et Diner (4 groupes de 6 menus), elle se repartit equitablement avec les Dejeuner et Diner
 d) _ On peut modifier/interchangee un menu journalier
      |-> ex: Petit-Dejeuner jour 1 -> Petit-Dejeuner jour 5 (et vice versa)
    _ On peut modifier/interchangee un menu hebdomadaire (autorisee seulement au debut du cycle)
      |-> ex: menu semaine 4 -> menu semaine 3
    _Seulement les menus de meme categorie peuvent s'echanger entre eux


------------------------------
2. REGLES DE CONCEPTION ET INTERFACE:
 a) Base de donnees: PostgreSQL
   -> Table a cree: * User -> l'utilisateur(consideree comme l'employe) qui doit se connecter pour acceder a l'application
                    * maraina -> liste recettes petit dej
                    * atoandro -> liste recettes dej
                    * hariva -> liste recettes diner
                    * boisson -> liste des boissons(avec petit dej)
                    * menu_maraina -> liste Petit-Dejeuner + boisson
                    * menu_journalier -> Petit-Dejeuner(menu_maraina) + Dejeuner + Diner
                    * diner_extra -> contenant les autres plat/recettes pour le diner (pas dans les 4 groupes)
 b) Affichage : _ Menu principale contenant :
                  |-> _ navbar
                      _ liens (en div): * Petit-Dejeuner(Petit-Dejeuner)
                                        * Dejeuner(Dejeuner)
                                        * Diner(Diner)
                                          |-> * Extra (Diner) : lien dans la page sur les 
                                        * Menu : liens vers la liste en tableau des menus hebdomadaire (tableau 1, tableau 2, ...) 
                                        * Statistique : affichant la liste des recettes deja cuisinee/fait avec certains informations (nombre de fois, date, ...) 
                      _ Menu de la semaine (un tableau contenant les menus journalier)
                _ Les menus journalier s'affichera en tableaux de 6 lignes(chaque ligne = 1 plan menu (Petit Dejeuner + Dejeuner +  Diner))
                   |-> - Chaque ligne auront un case a cocher(a cocher par l'utilsateir si deja cuisinee/fait) + bouton modifier
                       - bouton modifier = chaque menu peut etre modifier, si modif -> affichage liste deroulante des proposition
